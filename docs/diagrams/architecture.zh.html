codex-on-telegram — 架构与消息流

从 Telegram 继续本地 Codex 或 Claude Code 会话。一个所有者、一台 Mac — 完整往返：消息发进来，模型在你的项目里干活，回复与授权请求都回到聊天里。

Telegram（仅所有者）

  • • 私聊、群组与话题
  • • Bot API 长轮询拉取新消息
  • • 回复 + 内联授权按钮发回聊天
  • • 通过白名单锁定为你专属

桥接守护（共享核心）

  • • channel-router 把每个聊天映射到会话
  • • conversation-engine 流式处理对话
  • • delivery-layer 分段并重试回复
  • • 橙色 = 面向 Telegram 与文件的 I/O 边界

授权网关

  • • 工具调用在 canUseTool 处暂停
  • • 请求被推送到 Telegram
  • • 用按钮、1/2/3 或 /perm 批准
  • • 可在配置中开启自动批准

运行时 + 持久化

  • • CTI_RUNTIME 选择 Codex、Claude 或 auto
  • • 引擎在你的项目目录里运行 CLI
  • • 状态以 JSON 存于 ~/.claude-to-im
  • • 会话重启不丢 — 无需重新绑定

守护（macOS）

  • • launchd 让守护进程保持存活
  • • 健康检查在失败时重启
  • • 密钥只存 config.env，不入 plist
  • • 节流重启，避免崩溃循环
